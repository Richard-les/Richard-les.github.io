首先应该设计好todolist的格式，这里新采用了header、footer两个标签,这里使得网页的开头与结尾能更好的凸显出来并与用户输入的项目拼接在一起，格局更好看一些，另外使用了css的嵌套，虽然style中看起来麻烦，但是使用时无疑省了些事。另外cursor属性的加入也可以使用户直观的感受到哪些地方属于可点击项，opacity的不透明度则是使边框序列更好看些。

onclick的地方的设置直到后来才发现有错误，下面的js描述中并没有出现save用法，这里感觉header不用大费周章，我算是南辕北辙了。后面关于输入和删除的js相对比较简单，为了区分一键清除与某个事件的具体清除则使用了clear以及remove，添加事件中使用了trim的方法，可以去掉输入的空白项，使得输出的事件显得简单明了，这里if语句纯粹是为了防止他人输入空白，这样可以给予提醒，当然alert标签感觉很给力，push方法就是很简单的添加项主要用于数组后添加，unshift则相反用于在前面添加。

remove这里也犯了一些昏头的错误，使用splice后将指定某个事件抓取删除后从头删一个元素数组开始删应该就不用表明是0了。接下来update指令则是完成一项事件后，首先用splice在代办中拿出这一个事项，然后再使用splice将它添加进已办中。接下来的edit中就是将输入的值存储到名为title的值中。edit中使用p-以及input-能精确对某个事件进行编辑另外setSelectionRange使得点击后整个语句可被选中，onblur标签使得失去焦点后以下的函数就被调用。

load中使用for语句确保所有选项都能被遍历，使用【】来抽取要编辑要编辑的事项，draggable使事件能被拖动，checkbox与checked使已办事项能被勾选标记出来onchange调用update标签使得勾选的已办事项可以回到待办事项。这里还多使用了+=的html用法，例如a+=b相当于a=a+b。

animation做时钟的使用

经过使用javascript做出最简单的电子时钟，我进行了下一步尝试–指针时钟，说来简单，实则困难，首先表盘的制作与设计就比较困难，我最终选择先添加一个有边框的圆加上6条均匀穿过圆的直线，在里层再覆盖一个白色圆，这样刻度就可以被标出来，然后中心添加黑点附带指针就好，其中为了图片的平移，transform、translate指令自然不可少，正百分率代表向右与向下。

接下来就是重头戏了，使用animation让表转起来，首先使用animation自然要先明确关键帧的存在与命名，将每个需要运动的属性中加入animation指令开头明确名称，这里我用了duration、timing-function以及iteration-count来修饰每个动画。

钟表与电子时钟最大的不同在于钟表的指针，为了使指针形象，我在js中的if语句就得改善，为了使时针不呢么死板，我加入了math.floor（）属性来使时针转动更逼真，其中添加的函数则是以分针为基础的。

但我在制作中也碰到了些困难，当我用js获取本地时间后，我原本的animation动画使我的表盘无法从本地时间运行，只能从零点运行，经过Google与多次尝试，我发现给animation添加注释块就能正常运转，但注释块无意义，我的钟表依然能正常运动，我也很迷惑，希望学长学姐帮我康康，看是哪里的问题

javascript

js中的对象、函数、构造函数、实例对象以及函数与对象的关系

对象的概念就是对象就是一组“键值对”（key-value）的集合，是一种无序的复合数据集合。在代码中，每当声明变量后的大括号里就定义了一个对象，它被赋值给了声明的变量，所以该变量就指向了一个对象，对象内部可以继续添加键值对，前面的叫键名，冒号后的为键值。键名可以是数值也可以是字母，如果键名不符合标识名的条件（比如第一个字符为数字，或者含有空格或运算符），且也不是数字，则必须加上引号，否则会报错。对象每一个键名称为属性，如果属性的值为函数，就称属性为方法。属性可以动态创建不必在对象声明时就指定。在对象中还有一个比较特殊的存在叫实例对象，对象有专属于自己的属性，还有一个[prototype]指向可以通过prototype访问；。

我觉得在说对象方法前，有必要先了解一下函数的定义。函数是一段可以反复调用的代码块。函数还能接受输入的参数，不同的参数会返回不同的值。函数的声明总共有三种方法，第一种是function命令，function后是函数名，函数名后面是一对圆括号，里面是传入函数的参数。函数体放在大括号里面。第二种就是使用函数表达式，将一个匿名函数赋值给变量。这时，这个匿名函数又称函数表达式，因为赋值语句的等号右侧只能放表达式。采用函数表达式声明函数时，function命令后面不带有函数名。如果加上函数名，该函数名只在函数体内部有效，在函数体外部无效。第三种就是使用function构造函数，你可以传递任意数量的参数给Function构造函数，只有最后一个参数会被当做函数体，如果只有一个参数，该参数就是函数体。但这种方法不够直观也比较麻烦，所以不常使用。调用函数时，要使用圆括号运算符。圆括号之中，可以加入函数的参数。函数体内部的return语句，表示返回。JavaScript 引擎遇到return语句，就直接返回return后面的那个表达式的值，后面即使还有语句，也不会得到执行。也就是说，return语句所带的那个表达式，就是函数的返回值。return语句不是必需的，如果没有的话，该函数就不返回任何值，或者说返回undefined。

下来就说说对象方法，JavaScript 方法是能够在对象上执行的动作，JavaScript 方法是包含函数定义的属性。在 JavaScript 中，被称为 this 的事物，指的是拥有该 JavaScript 代码的对象，this 的值，在函数中使用时，是“拥有”该函数的对象，请注意 this 并非变量，它是关键词，无法改变 this 的值。

说到函数与对象的关系，我认为有必要先知道，在js中，一切都是对象，函数也是对象，对象都是由函数创建的。

css

css知识点总结

最近刚刚完成了css简单网页制作，眼过千遍不如手过一遍，在制作后又对css展开了一些深入，了解了不少新的内容，使我自己对于css制作更加了解，今天我就总结一下关于css内联、选择器、权重以及引用方面的知识。

首先我们说说内联。css的内联体现在两方面，第一个方面就是样式表方面的内联，又称为行内式，这种主要用于使用style标签将许多修饰词语用在一起，方便编写不需要写每个修饰词的title，同样用于样式表的还有两种格式，一种是内嵌样式（将修饰词写于head中的style）一种是外链式（需要用到css与html文件的抽离）。第二个方面则是元素方面，css中标签元素大体分为三种，一种就叫内联元素（ps：a、span、br…）其他两种为块状元素（div、p、h1…)与内联块状元素（img、input）。

第二点我们就要讲一讲选择器的部分，css中的选择器多的数不胜数，下面我将进行分类介绍。首先就是.class与#id，二者都具有引用的作用，class是一种样式，它可以出现在任何结构和内容上，并且多次使用，而id则像身份认证一样，只能使用一次，并且id可作为锚点出现作为链接。其次就是element选择器的多种使用，例如div与p，div p代表div结构以及p中的内容都将被修饰，div+p就代表div中的内容以及div后紧跟的p内容将被修饰。接下来就是一些网站的链接选择器，link标签指向的是我们从未访问的网站，而我们访问过的则可以使用visited来指向，如果想让我们的指向标签好看，就需要用到hover与active选择器（切记active永远在hover之后）。接下来就是一些我没用过的选择器，第一个就是focus，它更多用于被点击的目标，网页中有输入选项就可以使用这个。before与after选择器就是要在写的内容前方或者后方添加内容当我们不想使用class来指向修饰p部分的话，only-child,nth-child(n),root,last-child都将会是不错的选择。如果是需要输入的界面，那么说不定会有一些非法输入，这时就需要一些提示来提示输入者，例如invalid选择器，输入非法则会显示醒目的设置颜色。

关于权重问题我们首先要了解权重的五个等级，重要程度依次是！important、行内样式、id选择器、class、标签选择器和伪元素选择器。权重规则是这样的!important 优先级最高，但也会被权重高的important所覆盖，行内样式总会覆盖外部样式表的任何样式(除了!important)，单独使用一个选择器的时候，不能跨等级使css规则生效，如果两个权重不同的选择器作用在同一元素上，权重值高的css规则生效，如果两个相同权重的选择器作用在同一元素上：以后面出现的选择器为最后规则，权重相同时，与元素距离近的选择器生效。

最后就是引用问题了，引用分为行内式（在HTML标签中，使用style属性设置CSS样式）、嵌入式（将CSS样式集中写在HTML页面的style标签中）、导入式、链接式，后面两者都是从外部引用，但链接式对于客户端用户浏览网站时效果更好。

understanding

html、css做个人简介的小心得

我也是前端萌新，一周前第一次听到前端这种计算机概念，在一周的学习中我通过runnob网站学习了大概的一些概念，下面就是我在制作简介时的一些感受与学到的技巧。

刚开始做这个的确很不容易，首先得了解html结构，每一个属性标签的开启就会有相应的结束标签，这是最最基础的。对于无js基础的我style标签就成了我编辑最常用的工具，无论背景还是颜色还是字体大小都是通过style完成的（根据后来学长指导，抽离css与html会让文件干净并方便修改）。为了让文字出现在图片与表格中的位置合适，对于css盒子模型的了解是不可或缺的，cover与no repeat使背景图片充满浏览器页面，这时为了使文字出现在合适的位置，magin、border、padding可以使文字出现在任何位置（只要距离得当）。

下面，最重重重的要的事情来了，为了让自我介绍条理分明，导航栏总得要有，为了实现链接，就先使用标签定义链接，href指令可以连接到链接目标，这就是导航栏重要的部分，ul指令只是设置导航栏的位置与大小，为了让我的目录点击感更强，hover指令与active可以让目录点击感更加强烈也很漂亮（ps：active肯定是在hover之后的）。为了让页面根据我的目录更新，div块级元素是必须要有的，这个相当于在大盒子中创造一个单独的小盒子，学长向我推荐了一种很不错的方法就是在页面内设置锚点后用display来设置不同页面在哪种情况下显示，不显示的自然是hidden指令。经过我能偷懒的特性我选择了id指向不同链接的方法这样可以不需要设置display形式，当然id形式也是每个id指令都需要div再创建块级元素，这样输入自己的内容就很容易了。

剩下基础的内容设置就只要敲敲键盘打打字就好了，以上就是我的一些小见解。
